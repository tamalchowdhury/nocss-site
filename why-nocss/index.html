Why No CSS?

I was going through Twitter and found a tweet about No CSS.

People are always debating between Bootstrap and Tailwind for CSS...

But have you tried No CSS? pic.twitter.com/Yyg4ilECXK

— Sid Palas (@sidpalas) March 26, 2022
^this is the only thing in this website with CSS (loaded from Twitter)

Having a restriction can help us be creative.

Since I can't write any external or inline CSS,
I have to rely on default browser styles.

Font changes are not an option since you have to control that with CSS.

I'm missing my favorite Arial font:

    body { font-family: Arial, sans-serif; }

Working with just the vanilla HTML can actually teach you
about the default browser styles.

Notice, I am keeping my paragraphs short?

I don't want my text to span all across the wide screen.

Speaking of screens, @media queries are not possible.

What I build has to be optimized for both mobile and desktop.

Browser adds a default margin of 8px in body,
so the text doesn't touch the side of the screen.
